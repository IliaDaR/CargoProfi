Что внутри

GPS-трекинг рейсов

Водитель нажимает «Начать рейс» — система фиксирует координаты каждую минуту. Пробег считается по формуле гаверсинусов: сумма расстояний между всеми GPS-точками. Данные пишутся в Firestore и доступны владельцу в реальном времени. При обрыве связи точки копятся локально и синхронизируются при подключении.

Формула гаверсинусов Раз в 60 сек Offline-буфер

Путевые листы

Генерация PDF по форме, утверждённой Приказом Минтранса России. Владелец нажимает одну кнопку — и готовый документ с датой, госномером, ФИО водителя, пробегом, временем выезда и возврата. Все поля подставляются из данных рейса автоматически.

PDF одним кликом Приказ Минтранса Автозаполнение

Учёт расходов

Водитель добавляет расход прямо во время рейса: сумма, категория (топливо, стоянка, ремонт, дорожный сбор, мойка, шины, страховка, прочее) и фото чека. GPS-координаты и время прикрепляются автоматически. Владелец видит сводную таблицу по категориям и может открыть фото любого чека.

8 категорий Фото чеков GPS-метка Сводка

Расчёт зарплаты

Владелец задаёт правило для каждого водителя: процент от дохода рейса или фиксированная сумма за рейс. Расчёт зарплаты за любой период — один клик. Система соберёт все завершённые рейсы водителя, вычислит сумму и сохранит в ведомость с историей.

% или фикс За период Ведомость История

Контроль автопарка

Все машины на одном дашборде с индикатором статуса: «в рейсе» или «свободен». Владелец видит историю рейсов по каждому автомобилю, общий пробег, статистику использования. Добавление новой машины — госномер, марка, модель, год и VIN.

Статус онлайн История Статистика

Ролевой доступ

Водитель видит только свои рейсы и расходы. Владелец — все данные по парку. Каждый запрос к базе проверяется правилами Firestore на уровне сервера. Авторизация через Firebase Authentication: email/пароль или телефон.

Firebase Auth Роли owner/driver Server-side rules
